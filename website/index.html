Side-by-Side Comparison

See the difference between squircles and standard border-radius

Squircle

Continuous curve (smoothing: 0.8)

Border-Radius

Standard CSS border-radius

Squircles use a superellipse formula to create smoother, more continuous curves than standard border-radius. The smoothing parameter (0.0-1.0) controls how closely the curve follows the iOS-style continuous curve characteristic.

Tier Hierarchy

  1. Native CSS - GPU-accelerated, zero JavaScript overhead (Chrome 139+)
  2. Houdini Paint API - Runs on paint thread, near-native performance (Chrome 65+, Edge 79+)
  3. SVG ClipPath - Dynamic path generation, excellent performance (All modern browsers)
  4. Border-Radius Fallback - Standard rounded corners, universal compatibility

Code Examples & Installation

Installation

npm install @cornerkit/core

Vanilla JavaScript

import CornerKit from '@cornerkit/core';

// Create instance
const ck = new CornerKit();

// Apply to element
ck.apply('#my-element', {
  radius: 20,
  smoothing: 0.8
});

HTML Data Attributes

<div
  data-squircle
  data-squircle-radius="20"
  data-squircle-smoothing="0.8"
>
  Your content
</div>

<script>
  CornerKit.auto();
</script>

TypeScript

import CornerKit, { type SquircleConfig } from '@cornerkit/core';

const ck = new CornerKit();
const config: SquircleConfig = {
  radius: 20,
  smoothing: 0.8
};
ck.apply('#my-element', config);
